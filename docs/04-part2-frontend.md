### 2.2 左侧：参考资料坞 (Reference Dock)

固定在页面左侧，宽度 300px，包含以下模块：
1. **标签上下文 (Context)**: 列出用户在进入本页前所选的所有标签，作为整个配方生成的限制条件和依据。
2. **冰山理论分析**: 直接以纯文本形式展示该博主粉丝群体的“显性行为层”、“情感价值层”和“深层需求”，方便调香师随时查阅找寻灵感。
3. **原始画像报告入口**: 提供一个按钮，点击后可在当前页面弹出一个 Drawer 展示 Part 1 的原始雷达图和饼图数据。

### 2.3 中间：配方画布区 (Formulation Canvas)

#### 方案卡片设计

采用极简的纸质账本风格，1px 细边框，大面积留白。

```text
┌─ 方案一 ──────────────────────────────────────────────┐
│                                                        │
│  粉色之梦 — 花果甜香                           [Plan 1] │
│                                                        │
│  ── 前调 ──────────────────────────────────────────    │
│  佛手柑                                                │
│  明亮清新的开场                                         │
│  ↳ 推荐理由: 呼应粉色系审美中的活力感                  │
│                                                        │
│  ── 中调 ──────────────────────────────────────────    │
│  鸢尾花                                                │
│  粉质的优雅花香                                         │
│  ↳ 推荐理由: 连接古典系审美与蓝紫色偏好                │
│                                                        │
│  ── 后调 ──────────────────────────────────────────    │
│  沉香  [NEW]                                           │
│  深沉的东方木质                                         │

│                                                        │
└────────────────────────────────────────────────────────┘
```

**关键交互**:
1. **香材项**：每个香材显示名称、描述、推荐理由。
2. **变更高亮**：被对话修改的香材标注 `[NEW]` 文字徽标，并伴随背景色微弱闪烁动画。
3. **创作灵感**：展开后以引用块样式展示，用斜体排版，营造故事氛围。

#### 方案卡片变更动画

当右侧对话区 AI 返回了 `updated_plans` 时：
1. 卡片边框闪烁一次高亮色（如琥珀色）。
2. 被修改的香材项旧值淡出，新值淡入并显示 `[NEW]`。
3. 如果修改了推荐原因，面板自动展开。

### 2.4 右侧：AI 助手对话区

宽度固定 400px，提供与调香助手的多轮对话能力。

**消息渲染规则**:
- **用户消息**: 右对齐，琥珀色深色背景气泡。
- **助手消息**: 左对齐，浅色背景气泡配极细边框。
- **方案更新通知**: 如果 `updatedPlans` 不为空，在消息底部显示一个可点击的标签：`[方案 X 已更新，点击查看]`。点击后中间画布自动滚动到对应方案。

**快捷操作建议**:
输入框上方提供纯文本按钮（无 Emoji）：
`[换一种风格] [更浓郁] [更清淡] [适合夏天] [适合约会] [解释选择理由]`

### 2.6 沉浸式加载体验 (Embedded Loading State)

由于大模型生成需要数十秒的等待时间，为了避免全屏加载带来的“割裂感”与“等待焦虑”，本页面采用**三栏布局内嵌加载**的无缝过渡设计。

1. **初始状态 (Processing)**：
   - 顶栏与左侧的参考资料坞（已选标签）正常渲染。
   - **中间画布**：显示纯代码实现的“精油萃取”极简动画（精油滴落泛起涟漪），下方配合类似终端打字机效果的动态提取日志（如 `正在解析潜意识需求...`，`正在重组香调库...`）。
   - **右侧助手区**：输入框锁定。

2. **状态流转机制**：
   - **全程驻留**：如果用户一直停留在该页，当后端返回 `completed` 状态时，中间的萃取动画会平滑消散，紧接着左侧冰山分析淡入，中间的多套方案卡片以**“阶梯式（Staggered Fade-Up）”**从下往上平滑浮现。
   - **切出后切回**：如果用户退回 Dashboard 后再点击 `[进入调配室]`，若状态已是 `completed`，则**直接跳过长时间的加载动画**，瞬间播放 1 秒钟的“阶梯式卡片浮现入场动画”，既保证高级感又避免多余等待。

### 2.5 顶部工具栏

- **返回首页工作台**: 退出调配室，回到 Dashboard。
- **重新设置标签**: 弹窗警告：“这将暂停当前会话，回到标签筛选页重新创建调配方案。”确认后跳转 `/tags/:taskId`。

## 三、数据流

### 3.1 页面初始化

```
进入页面 (带 sessionId 参数)
    ↓
调用 GET /api/v1/fragrance/{sessionId}
    → 获取推荐方案 + 标签上下文 + 冰山分析数据
    ↓
调用 GET /api/v1/fragrance/{sessionId}/history
    → 获取对话历史
    ↓
渲染三栏工作台
```

### 3.2 对话修改方案

```
用户输入消息 → 点击发送
    ↓
调用 POST /api/v1/fragrance/{sessionId}/chat
    → 返回 { reply, updated_plans }
    ↓
添加 AI 回复到对话列表
    ↓
如果 updated_plans 不为空:
    → 更新中间方案画布
    → 触发局部淡入动画
    → 消息尾部附加链接
```

## 四、状态管理

### 4.1 Zustand Store (`useFragranceStore`)

管理全局的 `sessionId`，当前方案列表 `plans`，对话历史 `messages`，冰山分析数据 `icebergAnalysis`，已选标签 `selectedTags`，以及触发高亮动画的 `changeAnimation` 状态。

### 5.2 严格无 Emoji 政策

在所有的组件渲染中，包括 Mock 数据、API 返回的剧本、以及硬编码的 UI 文字，均不得出现任何系统自带的 Emoji。这确保了整个项目在视觉上的高度统一、高级和克制。
